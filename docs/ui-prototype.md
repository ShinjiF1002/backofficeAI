# バックオフィス自動化 管理画面プロトタイプ

> 本資料は、アーキテクチャ設計書で定義した仕組みを現場の担当者と管理者がどのように操作するかを、画面イメージで示したものです。プロトタイプ（画面構成の検討素材）であり、最終デザインではありません。

---

## この管理画面が実現すること

本システムの管理画面は、以下のサイクルを回すためのインターフェースです。

```
  AIが業務を実行する
       |
       v
  担当者がチェックし、承認する ─── 問題なければ完了
       |
       | 誤りがあった場合
       v
  担当者が修正コメントを残す
  「端数は四捨五入ではなく切り捨て」
       |
       v
  AIが学習し、同じミスを繰り返さなくなる
       |
       v
  精度が上がり、承認の手間が減る
       |
       v
  担当者の負荷が下がり、省力化が実現する
```

この資料では、**導入初日から3ヶ月後まで**、管理画面で何が起きるかを時系列で説明します。

---

## 導入初日: 手順定義の確認

AIに業務マニュアルを読み込ませると、AIが手順定義を自動生成します。管理者はこの内容を確認し、承認します。これが本システムとの最初の接点です。

## 1週目: 全ステップ承認で運用開始

手順定義が承認されると、AIが実際の業務を実行し始めます。初期は**すべてのステップで人間の承認が必要**です。担当者は毎日、ホーム画面から承認作業を行います。

## 1ヶ月目: 修正コメントが手順改善につながる

運用開始から1ヶ月が経つと、担当者の修正コメントが蓄積されます。AIはこれらのコメントを分析し、「手順そのものを変えたほうがよい」と判断した場合に変更提案を生成します。

**この提案は管理者が承認するまで反映されません。** どんなに軽微な変更でも、人間の承認なしにAIが勝手にルールを変えることはありません。


変更提案は「何を」「なぜ」「影響は」の3点で構成されています。技術的な表記（前後の差分表示など）は使わず、管理者が自然に読める形式です。

根拠として現場の修正コメントがそのまま引用されるため、「佐藤さんと田中さんが3回指摘している」という事実に基づいて承認判断ができます。

## 3ヶ月目: 信頼を獲得し、承認の手間が減る

精度が十分に向上した業務は、信頼レベルを昇格できます。昇格すると、重要なステップのみ承認が必要になり、担当者の負荷が下がります。

### 承認ステップの変化

```
 初期（全ステップ承認）:

  +-----+    +-----+    +-----+    +-----+    +-----+
  |承認 | -> |承認 | -> |承認 | -> |承認 | -> |承認 |
  |必要 |    |必要 |    |必要 |    |必要 |    |必要 |
  +-----+    +-----+    +-----+    +-----+    +-----+
   Step1      Step2      Step3      Step4      Step5

  担当者は5回の承認判断が必要


 昇格後（チェックポイント承認）:

  +-----+    +-----+    +-----+    +-----+    +-----+
  | 自動 | -> | 自動 | -> |承認 | -> | 自動 | -> |承認 |
  | 実行 |    | 実行 |    |必要 |    | 実行 |    |必要 |
  +-----+    +-----+    +-----+    +-----+    +-----+
   Step1      Step2      Step3      Step4      Step5

  担当者は2回の承認判断のみ（60%削減）


 十分な実績後（事後確認。高リスク業務は対象外）:

  +-----+    +-----+    +-----+    +-----+    +-----+
  | 自動 | -> | 自動 | -> | 自動 | -> | 自動 | -> | 自動 |
  | 実行 |    | 実行 |    | 実行 |    | 実行 |    | 実行 |
  +-----+    +-----+    +-----+    +-----+    +-----+
   Step1      Step2      Step3      Step4      Step5

  処理完了後にログで確認のみ
```

高額送金のような高リスク業務は、どれだけ実績を積んでも完全自動化しません。

## 監査対応

すべての画面での操作は自動的に記録されます。

```
  AIの操作記録                         人間の操作記録
  +-------------------------------+   +-------------------------------+
  | - 各ステップのスクリーン       |   | - 誰が、いつ承認/差し戻し    |
  |   ショット（操作前後）         |   |   したか                      |
  | - AIの判断理由                 |   | - 修正コメントの全文          |
  | - 参照した知見                 |   | - 変更提案の承認/却下記録     |
  | - 自動チェック結果             |   | - 信頼レベル昇格の承認記録    |
  +-------------------------------+   +-------------------------------+
               |                                    |
               +------------------+-----------------+
                                  |
                                  v
               「AIが何をして、人間が何を判断したか」を
               完全に追跡可能。監査時に「この処理は誰の
               責任で承認されたか」を特定できます。
```

## まとめ: この管理画面が支える4つの原則

| アーキテクチャ設計書の原則 | 管理画面での体現 |
|----------------------------|------------------|
| すべての変更は人間が承認する | 承認画面の承認ボタン。変更提案レビュー。AIが勝手に進めることはない |
| すべての操作は記録される | 全画面で操作を自動記録。スクリーンショットを含む完全な操作証跡 |
| 信頼は実績で獲得する | 学習状況画面の精度推移。客観的データに基づく段階的な自律化 |
| 使うほど賢くなる | 修正コメント -> 知見の蓄積 -> 精度向上。反映状況の可視化 |

本システムの管理画面は、現場担当者の業務知識をAIに安全に移転するためのインターフェースです。担当者が修正コメントとして残す暗黙知は、組織の知的資産としてシステムに蓄積されます。退職や異動で失われることなく、すべてのAIエージェントに即座に共有されます。
